================================================================================
USAGE EXAMPLES
================================================================================

1. Local Development Testing
   $ python testTTS.py --mode live --url ws://localhost:8000

2. Staging Pre-Deployment Validation
   $ export GOOGLE_API_KEY=your-key
   $ ./scripts/test_deployment.sh staging

3. Automated CI/CD Pipeline
   $ pytest tests/test_tts_smoke.py -v --base-url=https://staging.run.app

4. Production Health Check
   $ python testTTS.py --mode live --url wss://production.run.app --json-output

5. Custom Testing
   $ python testTTS.py \
       --mode live \
       --url ws://localhost:8000 \
       --voice kore \
       --text "Custom test message" \
       --timeout 60 \
       --output test-output.ogg

================================================================================
VALIDATION CRITERIA
================================================================================

The smoke tests validate the following:

Audio Metadata:
  ✓ MIME type matches expected format
  ✓ Sample rate = 24000 Hz
  ✓ Audio data is not empty
  ✓ File format signature is valid

Performance:
  ✓ TTFB < 500ms (warning threshold)
  ✓ Total duration < 30s (timeout)
  ✓ At least 1 audio chunk received
  ✓ Audio size > 1KB

WebSocket Protocol:
  ✓ Connection successful
  ✓ tts-hello message received
  ✓ Binary frames supported
  ✓ tts-done message with metadata

Health Checks:
  ✓ /health endpoint returns 200
  ✓ /system/tts-config returns valid JSON
  ✓ WebSocket endpoint accepts connections
