# Real-Time TTS Streaming Integration Plan

This document outlines a comprehensive, step-by-step plan to implement and validate real-time Text-to-Speech (TTS) streaming in your app using a WebSocket backend. Each step includes a checkbox to track progress. **Mark steps as complete as you finish them.**

---

## 1. Backend: WebSocket TTS Streaming Endpoint

- [x] **Implement `/ws/tts` WebSocket endpoint**
    - Accepts JSON requests with `text`, `voice`, and `params`.
    - Streams audio chunks as they are generated by the TTS engine (using OpenAI or similar).
    - Each chunk is base64-encoded and sent as `{ "type": "audio_chunk", "data": "..." }`.
    - Sends `{ "type": "done" }` at the end.
    - Handles errors and sends `{ "type": "error", "detail": "..." }` if needed.

- [x] **Implement `stream_speech` async generator in `VoiceService`**
    - Uses OpenAI's TTS streaming API via `aiohttp`.
    - Yields audio chunks as soon as they are available.
    - Handles correct `response_format` (e.g., `opus` as primary, `mp3` as fallback).

- [x] **Deploy and test backend endpoint**
    - Confirmed with `wscat` and a Python script that audio chunks are streamed and playable.

---

## 2. Frontend: Flutter App Integration

- [x] **Add WebSocket client for TTS streaming**
    - Use `web_socket_channel` or similar package.
    - Connect to `wss://ai-therapist-backend-385290373302.us-central1.run.app/voice/ws/tts`.
    - Handle connection, reconnection, and error states.

- [x] **Send TTS request with correct format**
    - Send a JSON message: `{ "text": "...", "voice": "...", "params": { "response_format": "opus" } }`.
    - Ensure `response_format` is supported by backend and TTS engine. If opus is not available, fallback to mp3 by sending `{ "response_format": "mp3" }`.

- [x] **Collect and decode audio chunks**
    - On receiving `{ "type": "audio_chunk", "data": "..." }`, base64-decode and append to a buffer.
    - On `{ "type": "done" }`, finalize the buffer.
    - Optionally, play audio as chunks arrive (for true streaming playback), or after all chunks are received.

- [x] **Play the audio**
    - Use a package like `just_audio` or `audioplayers`.
    - For streaming playback, use a custom audio source or stream buffer.
    - For post-collection playback, write buffer to a temp file and play.

- [x] **Handle errors and edge cases**
    - Show user-friendly error messages if the connection fails or backend returns an error.
    - Handle network interruptions and retries.
